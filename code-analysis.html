Modify existing test

Update login link visibility assertion to match new 'Sign In' label

frontend/tests/navigation.spec.ts

The app legitimately renamed the navbar login link from 'Login' to 'Sign In', so the existing test that looks for the exact text 'Login' will fail on every run until corrected. Fixing the selector keeps the regression suite green and continues to protect the real behaviour — that unauthenticated visitors can see and click a link to the login page.

Coverage check: navigation.spec.ts 'login link is visible when not logged in' is the only test asserting the navbar login link text. auth.spec.ts covers the login page itself but not the navbar link. No other test checks this label, so updating here is not a duplicate.

Current
  test('login link is visible when not logged in', async ({ page }) => {
    // Navbar shows "Login" (not "Sign In") when user is not authenticated
    const loginLink = page.getByRole('link', { name: /^Login$/i });
    await expect(loginLink).toBeVisible();

Proposed
  test('login link is visible when not logged in', async ({ page }) => {
    // Navbar shows "Sign In" (renamed from "Login") when user is not authenticated
    const loginLink = page.getByRole('link', { name: /^Sign In$/i }).first();
    await expect(loginLink).toBeVisible();
    await expect(loginLink).toHaveAttribute('href', '/login');
  });

New test

Mobile menu shows 'Sign In' link and navigates to login page

frontend/tests/navigation.spec.ts

The mobile menu login link was also renamed but is not covered by any existing test — only the desktop link was previously tested. Adding this test ensures mobile users can always find and use the login entry point, which is a critical conversion path on small-screen devices.

Coverage check: navigation.spec.ts covers the desktop navbar login link only; no test sets a mobile viewport or opens the hamburger menu to verify the mobile login link. This is not a duplicate of the desktop assertion.

View proposed test code ↕ 
import { test, expect } from '@playwright/test';

test.describe('Navigation — mobile menu', () => {
  test.use({ viewport: { width: 390, height: 844 } });

  test('mobile menu shows Sign In link that navigates to login page', async ({ page }) => {
    await page.goto('/');

    // Open the mobile hamburger menu
    const menuButton = page.getByRole('button', { name: /menu|open/i });
    await expect(menuButton).toBeVisible();
    await menuButton.click();

    // The mobile 'Sign In' link should be visible
    const signInLink = page.getByRole('link', { name: /^Sign In$/i });
    await expect(signInLink).toBeVisible();
    await expect(signInLink).toHaveAttribute('href', '/login');

    // Clicking it should navigate to the login page
    await signInLink.click();
    await expect(page).toHaveURL('/login');
  });
});
